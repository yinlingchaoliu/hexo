Posted on 2020-05-26 | In 重构-改善代码设计

原则：单一职责、最少知道原则

1）搬移函数
要点：使用频次高，搬移
2）搬移字段
要点：使用频次
3）提炼类
要点：某个类做了应该由两个类做的事
4）将类内联化
要点：某个类没做太多事
5）隐藏 “委托关系”
要点：客户通过一个委托类来调用另一个对象

要点：某个类做个多过简单委托动作
7）引入外加函数
要点：提炼一个函数减少变化

要点：新建一个类做扩展，为了安全
